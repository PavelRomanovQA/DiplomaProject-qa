# Планирование автоматизации

## _Перечень автоматизируемых сценариев_

### Позитивные проверки:
    Оплата тура дебетовой картой / кредитной картой

##### 1. Оплата тура по активной карте. Валидные данные.
* Шаги:
1. В поле "Номер карты" ввести "4444 4444 4444 4441". 
2. В поле "Год" ввести две последние цифры с текущей даты (например, "24").
3. В поле "Месяц" ввести число с текущей даты (например, "06").
4. В поле "Владелец" ввести имя и фамилию латинскими буквами (например, "Alex Ivanov"). 
5. В поле "CVC/CVV" ввести любые три цифры (например, "999"). 
6. Нажать кнопку "Продолжить". 
7. Запросить статус последней операции в базе данных app (в соответствующей СУБД).

_Ожидаемый результат:_

- Поведение веб-интерфейса: уведомление "Успешно! Операция одобрена банком" в правом верхнем углу.
- Статус операции в базе данных: "APPROVED".

##### 2.Проверка ввода в поле "Владелец" значения с тире.
* Шаги:
1. В поле "Владелец" ввести имя и фамилию латинскими буквами с тире (например, "Alex-Franc Ivanov").
2. Заполнить остальные поля валидными значениями.
3. Нажать кнопку "Продолжить".

_Ожидаемый результат:_
- Поведение веб-интерфейса: уведомление "Успешно! Операция одобрена банком" в правом верхнем углу.
- Статус операции в базе данных: "APPROVED".

##### 3.Проверка ввода в поле "Владелец" имени с сокращением.
* Шаги:
1. В поле "Владелец" ввести инициал имени и фамилию латинскими буквами, имя должно быть сокращенно с точкой (например, "A. Ivanov").
2. Заполнить остальные поля валидными значениями.
3. Нажать кнопку "Продолжить".

_Ожидаемый результат:_
- Поведение веб-интерфейса: уведомление "Успешно! Операция одобрена банком" в правом верхнем углу.
- Статус операции в базе данных: "APPROVED".

## _Перечень используемых инструментов_
- #### Все тесты будут написаны на Java, потому что и приложение, и инфраструктура написаны на этом языке.
  * IntelliJ IDEA. Платформа для удобного написания кода, в том числе для тестов.
  * JUnit. Библиотека для написания и запуска авто-тестов.
  * Gradle. Система управления зависимости, которая позволит удобно ставить необходимые фреймворки без проблем с постоянной настройкой зависимостей.
  * Selenide. Фреймворк для автоматизированного тестирования веб-приложений.
  * Allure. Фреймворк, предназначенный для создания подробных отчетов о выполнении тестов.
  * Docker -для работы с контейнерами.
  * DBeaver (работа с БД)
  * DevTools. Для просмотра кода элемента.
  * Git и Github. Система контроля версий, для хранения кодов автотестов и настроек окружения.

## _Перечень возможных рисков при автоматизации_
  * Могут возникнуть проблемы с правильной идентификацией полей ввода.
  * стоимость автоматизации тестирования.
  * Риск неработающего заявленного функционала приложения.
  * Может возникнуть проблема с настройкой приложения и БД.

## _Интервальная оценка с учетом рисков в часах_
* подготовка окружения, настройка и развертывания БД - 8 часов.
* написание автотестов, тестирование, отладка тестов - 48.
* формирование и анализ отчетов - 2 часа.
